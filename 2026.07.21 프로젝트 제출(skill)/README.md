# Project Sync 스킬

## 왜 만들었나

### 😩 불편했던 점
- 코드/파일 작업을 끝낼 때마다 "커밋해줘", "push 해줘"라고 매번 직접 말해야 했음
- PRD.md(기획/현황 문서)를 만들어놔도 실제 작업 내용이 반영되지 않고 방치되기 일쑤였음
- 무엇보다, 스킬을 만들어놔도 "이 정도는 그냥 git 명령어 몇 줄이면 되니까 스킬까지 쓸 필요는 없겠다"는 판단 때문에 가끔 자동으로 실행되지 않는 경우가 있었음 (트리거가 들쭉날쭉함)

### ✅ 스킬이 어떻게 없앴나
| 불편했던 점 | 스킬의 해결 방식 |
|---|---|
| 매번 커밋/push를 직접 요청해야 함 | 프로젝트 폴더에서 파일 작업(생성/수정/삭제)이 끝나면 요청 없이 자동으로 커밋 + GitHub push까지 처리 |
| PRD.md가 최신 상태로 안 유지됨 | 커밋 전에 PRD.md 전체를 프로젝트의 "현재 상태"에 맞춰 다시 작성 (변경 로그 누적이 아니라 살아있는 문서로 유지) |
| "간단해 보여서" 스킬이 가끔 안 켜짐 | description에 "커밋 정도는 내가 그냥 하면 되지"라는 판단이 바로 이 스킬을 건너뛰게 만드는 함정이라는 걸 못박아두고, 실행을 건너뛰어도 되는 경우를 **딱 3가지(작업 미완성 / 사용자의 명시적 보류 요청 / 프로젝트와 무관한 일회성 작업)**로 한정해서, 그 외에는 예외 없이 실행되도록 설계함 |

## 파일 구성
- `SKILL.md` — 스킬 본문 (트리거 조건 + 실행 절차, 읽기용)
- `project-sync.skill` — Claude에 바로 설치할 수 있는 패키지 파일 (SKILL.md + references + scripts 포함)

## 설치 방법
`project-sync.skill` 파일을 Claude와의 대화에 업로드하면 파일 카드에 **"Save skill"** 버튼이 뜨고, 클릭하면 프로필에 스킬이 설치됩니다.
또는 개인설정 스킬 항목에서 .skill 파일을 직접 업로드합니다.
